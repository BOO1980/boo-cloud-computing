Cloud Computing Overview

Infrastructure as a service

  • Infrastructure as a service (IaaS) is the most flexible category of cloud services
  • It provides you the maxium amount of control for your cloud resources
  • In an LaaS model, the cloud provider is responsible for maintaining the
    • hardware
    • network connectivity
    • physical security
  • Youre responsible for everything else
    • Operating system installation
    • Configuration
    • Maintenance
    • Network Configuration
    • Database and storage configuration
  • With IaaS, your essentially renting the hardware in a cloud datacenter
  • With all the other cloud service types IaaS places the largest share of the responsibilty with you
  • Some common scenarios where IaaS might make sense include
    • Lift and shift migration: Your'e standing up cloud resources similar to your on-prem datacenter, and then simily moving the things runnon on-prem to running on the IaaS infrastructure
    • Testing and development: You have established configuration for development and test environments that you need too rapidly replicate, You can stand up or shut down the different enviroments rapidly with an IaaS struture, while maintaining compelte control

Platform as a Service

  • Platform as a service (PaaS) is a middle ground between renting space in a datacenter (Infrastructure as a Service) and paying for a complete and deployed solution (Software as a Service).
  • In a Paas environment the cloud provider maintains the physical infrastructure, physical securitym and connection to the internet
  • They also maintain the operatinf systems, middleware, development tools and buisness intelligence services that make up a cloud solution
  • In a Paas environment you dont have to worry about the licensing or patching for operating systems and databases
  • Paas is well suited to provide a complete development enviroments without the headache of maintaining all the developemtn infrastructure
  • In a PaaS splits the responsibilty between you and the cloud provider
  • The cloud provider is responsible for maintiaing the physical infrastructure and its access to the internet
  • Think of PaaS like using a domain joined machine: IT maintains the device with regular updates patches and refreshes
  • Common scenarios where PaaS might make sense is
    • Development Framework: PaaS provides a framwork that developers can build upon to develop or customize cloud-based applications
    • PaaS lets developers create applications using built in software components
    • Analytics or business intelliegence: Tools provided as a service with PaaS allow organisations to analyze and mine their data, fnding insights, and patterns and predicting outcomes to improve forecasting, product design decisions, investment returns and other business decisions
